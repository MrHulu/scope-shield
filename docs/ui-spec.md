# Scope Shield — UI 规格

## 范围

MVP 的页面清单、信息架构、每页行为、状态处理、响应式规则、两套视觉风格定义。

## 非范围

- 移动端适配
- 主题切换（深色模式）
- 动效细节（Lottie 级）

## 假设

- 桌面浏览器为主（≥ 1024px）
- App 界面 = 中性风格（Linear/Notion）
- 导出图片 = Apple/iOS 亮色风格（独立于 App）

## 依赖

- docs/mvp-prd.md 功能需求
- docs/flows.md 核心流程和状态机
- docs/data-model.md 实体定义

## 信息架构

```
Scope Shield
├─ [左侧边栏] 项目列表
│   ├─ 进行中（默认展开）
│   │   ├─ 项目 A
│   │   ├─ 项目 B
│   │   └─ + 新建项目
│   ├─ 已归档（折叠）
│   │   └─ 项目 C
│   └─ ⚙️ 设置（底部固定）
│
└─ [主内容区] 项目详情
    ├─ 项目头部（名称 + 统计卡片）
    ├─ 需求列表区
    │   ├─ 需求行 × N
    │   └─ + 添加需求
    ├─ 图表区
    │   ├─ Tab: 简洁版 | 详细版
    │   ├─ 图表渲染
    │   └─ 导出图片按钮
    └─ 变更记录区
        ├─ 变更行 × N
        └─ + 记录变更
```

## 页面清单

| 页面/面板 | 路由 | 说明 |
|-----------|------|------|
| 项目详情页 | `/project/:id` | 主界面，含需求+图表+变更 |
| 设置页 | `/settings` | JSON 导入/导出 |
| （无独立页面） | — | 新建项目、记录变更等均为弹窗/内联 |

**路由规则**：
- `/` → 重定向到最近访问的项目（或 Demo 项目）
- `/project/:id` → 项目详情（ID 不存在时重定向到最近有效项目或 Demo）
- `/settings` → 设置页
- 其他路径 → 重定向到 `/`

**说明**：MVP 采用单页架构，核心交互集中在项目详情页，通过弹窗和内联表单完成所有操作。

## 组件规格

### 侧边栏

| 元素 | 行为 |
|------|------|
| 项目列表 | 分"进行中"和"已归档"两组，各组可折叠 |
| 项目项 | 点击切换主内容区，当前项目高亮 |
| + 新建项目 | 内联输入框（名称+日期），回车或按钮保存 |
| Demo 标记 | Demo 项目名称旁显示小标签 |
| 归档分组 | 默认折叠，点击展开 |

**状态**：
- 默认：显示项目名称
- 选中：背景高亮 + 左侧指示条
- 空：显示"新建项目开始使用"

### 项目头部

| 元素 | 说明 |
|------|------|
| 项目名称 | 可编辑，点击进入编辑态 |
| 统计卡片组 | 横向排列 5 个卡片 |

**统计卡片**：

| 卡片 | 数据 | 格式 |
|------|------|------|
| 原始工期 | originalTotalDays | "X 天" |
| 当前工期 | currentTotalDays | "X 天" |
| 膨胀率 | inflationRate | "+X%" 红色 / "0%" 灰色 / "-X%" 绿色 |
| 变更次数 | totalChanges | "X 次" |
| 补充次数 | supplementCount | "X 次"（"温水煮青蛙"指标，含 daysDelta=0 的补充记录） |

### 需求列表

| 元素 | 行为 |
|------|------|
| 需求行 | 名称 + 天数 + 状态标记 + 依赖标签 |
| 拖拽手柄 | 左侧，用于调整显示顺序（拖拽直接更新 sortOrder，不创建变更记录。变更弹窗中的"调优先级"类型会同时记录变更） |
| 操作菜单 | 悬停显示：编辑、设置依赖、删除 |
| 编辑需求 | 点击后需求行就地展开为内联编辑表单（与添加需求相同布局）。可编辑字段：名称、天数、依赖。保存后根据是否有变更记录决定是否触发 replay（见 FR-11） |
| + 添加需求 | 内联表单展开：名称输入 + 天数输入 + 依赖下拉(可选) |

**需求行状态显示**：

| 状态 | 视觉 |
|------|------|
| active | 正常文字 |
| paused | 灰色文字 + 暂停图标 |
| cancelled | 灰色文字 + 删除线 + "节省 X 天" 绿色标签 |

### 图表区域

| 元素 | 行为 |
|------|------|
| Tab 栏 | "简洁版" / "详细版"，默认简洁版 |
| 图表画布 | SVG 渲染，实时更新 |
| 导出按钮 | 右上角，点击弹出尺寸选择 |

**图表空状态**：无需求且无变更时显示"添加需求后即可看到图表"，导出按钮置灰。**特殊情况**：无需求但有变更记录（需求全部硬删除）时，简洁版显示变更段（标注"已删除"）且可导出，详细版时间线显示变更事件。

### 变更记录区

| 元素 | 行为 |
|------|------|
| 变更行 | 日期 + 类型图标 + 描述 + 角色/人名 + 天数影响 |
| 操作 | 悬停显示编辑/删除按钮 |
| + 记录变更 | 打开变更弹窗 |
| 排序 | **列表**：按日期倒序（最新在上，方便查看最近变更）。**图表段**：按日期正序（最早在左，从左到右时间顺序排列） |

### 变更弹窗（核心交互，30秒设计）

```
┌─────────────────────────────────────┐
│ 记录变更                        ✕   │
├─────────────────────────────────────┤
│                                     │
│ 变更类型（7 选 1，大按钮/卡片）      │
│ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐  │
│ │加天数│ │新需求│ │砍需求│ │补充  │  │
│ └─────┘ └─────┘ └─────┘ └─────┘  │
│ ┌─────┐ ┌─────┐ ┌─────┐           │
│ │调优先│ │ 暂停 │ │ 恢复 │           │
│ └─────┘ └─────┘ └─────┘           │
│                                     │
│ ── 动态字段（根据类型显示）──        │
│                                     │
│ 角色  [产品经理 ▼]  人名 [      ]   │
│ 描述  [                         ]   │
│ 日期  [今天 ▼]                      │
│                                     │
│              [取消]  [保存]          │
└─────────────────────────────────────┘
```

**类型 → 动态字段映射**：

| 类型 | 额外字段 |
|------|----------|
| 加天数 | 选择需求(下拉，仅 active) + 增加天数(数字) |
| 新增需求 | 需求名称(文本) + 天数(数字) |
| 砍需求 | 选择需求(下拉，仅 active) |
| 调优先级 | 选择需求(下拉) + 新位置(1-based 序号下拉，如"第 1 位"/"第 2 位"…)。**注意**：UI 显示 1-based，存入 metadata.fromPosition/toPosition 为 0-based（UI 值 - 1） |
| 需求补充 | 选择需求(下拉，所有状态均可) + 补充子类型(3选1：功能补充/条件变更/细节细化) + 天数(数字，允许0，支持0.5粒度) + 描述(必填) |
| 暂停 | 选择需求(下拉，仅 active) + 剩余天数(数字，默认=currentDays) |
| 恢复 | 选择需求(下拉，仅 paused) |

**需求下拉过滤规则**（新建和编辑弹窗通用）：
- `add_days`：仅 active 需求
- `cancel_requirement`：仅 active 需求（AC-58/AC-114）
- `pause`：仅 active 需求
- `resume`：仅 paused 需求
- `supplement`：所有 active + paused + cancelled 需求（不受 status 限制，记录的是"需求范围变更的事实"）
- `reprioritize`：所有 active + paused 需求（cancelled 除外）
- 编辑弹窗中切换 type 时，target 下拉实时按新 type 过滤

**30秒保障设计**：
- 类型选择用大按钮（一点即选），非下拉菜单
- 角色默认"产品经理"，多数情况不需改
- 人名有历史建议（下拉自动补全）
- 日期默认今天，多数情况不需改
- 描述单行输入，不需长文
- 共 3-4 次点击/输入即可完成

### 编辑变更弹窗

复用「记录变更」弹窗布局，预填已有数据。区别：
- 标题改为"编辑变更"
- 类型按钮区显示当前类型（已选中态），`new_requirement` 类型的 type 按钮不可更改
- `new_requirement` 类型的关联需求不可更改（字段置灰）
- 其他字段均可编辑
- type 更改时动态字段联动：改为 `cancel_requirement` 时 daysDelta 隐藏（系统自动计算）；改为 `pause` 时显示"剩余天数"；改为 `resume`/`reprioritize` 时 daysDelta 隐藏；改为 `add_days` 时显示 daysDelta 输入框；改为 `supplement` 时显示补充子类型(3选1) + daysDelta 输入框(允许0) + 描述(必填)
- 修改 type/target/daysDelta/date/metadata 中影响计算的字段时触发 replay 重算；仅修改描述/角色/人名则直接更新，不触发 replay

## 状态处理

### 全局状态

| 状态 | 显示 |
|------|------|
| 首次加载（无数据） | 自动创建 Demo 项目，跳转到 Demo 项目详情 |
| 正常使用 | 侧边栏 + 主内容区 |
| IndexedDB 不可用 | 全屏错误提示"请使用支持 IndexedDB 的浏览器" |

### 项目详情状态

| 状态 | 显示 |
|------|------|
| 有需求 + 有变更 | 完整界面，图表渲染 |
| 有需求 + 无变更 | 简洁版：仅显示一条蓝色原始计划条（不显示实际工期对比条），底部显示"原始计划 X 天 · 暂无变更"（灰色）。详细版：时间线区显示"暂无变更记录"，甘特正常渲染纯蓝色条，底部显示"原计划 X天 → 实际 X天 · 暂无变更"。变更列表区显示"暂无变更记录" |
| 无需求 | 需求区引导文案，图表区引导文案，导出按钮置灰 |
| 已归档项目 | 所有编辑/添加/变更按钮隐藏，仅保留查看和导出 |

### 表单验证

| 字段 | 规则 | 错误反馈 |
|------|------|----------|
| 项目名称 | 非空，≤ 100 字符 | 红色边框 + "请输入项目名称" |
| 需求名称 | 非空，≤ 200 字符 | 红色边框 + "请输入需求名称" |
| 天数（需求/变更） | 正数，≥ 0.5，支持 0.5 步进（0.5, 1, 1.5, 2...） | 红色边框 + "请输入有效天数（最小 0.5）" |
| 补充天数 | 数字，≥ 0，支持 0.5 步进（0, 0.5, 1, 1.5...） | 红色边框 + "请输入有效天数（允许 0）" |

| 变更描述 | 非空，≤ 500 字符 | 红色边框 + "请填写变更描述" |

## 图表渲染规格

### 简洁版（膨胀对比条）

```
原始计划  ████████████████  15天

实际工期  ████████████████ ▓▓▓▓ ▒▒▒ ░░  24天
          原始15天    PM+4  领导+3  测试+2

延期 9 天 = 100% 来自需求变更
膨胀率 +60%
```

- 上方：蓝色原始计划条
- 下方：蓝色原始 + 分段着色（每次变更一段）
- 颜色按角色：PM=红, 领导=橙, 测试=紫, 其他=灰。**例外**：`new_requirement` 类型段使用靛色（App: #5B21B6 / 导出: #5856D6），不按角色着色，与 QA 紫色（#7C3AED / #AF52DE）区分；`supplement` 类型段使用玫瑰红（#E11D48），不按角色着色
- 变更段按时间顺序从左到右排列（与变更记录的 date 排序一致）
- 每段标注：变更日期 + 角色 + 人名(可选) + 描述
- 砍需求：绿色段，标注"节省 X 天"
- **需求补充(supplement)**：daysDelta > 0 时在膨胀条中渲染玫瑰红段（#E11D48），标注"补充 +X天"；daysDelta = 0 时不在膨胀条中渲染段（但计入 changeCount 和补充次数统计）
- **暂停/恢复**：daysDelta=0，不在膨胀条中渲染段（仅在详细版时间线中记录事件）
- **调优先级**：daysDelta=0，不在膨胀条中渲染段
- 底部总结区域（固定格式）：
  - 第一行：`延期 {totalDelay} 天`（大字号，红色）。totalDelay = 0 时显示"延期 0 天"（灰色）；totalDelay < 0 时显示"提前 {|totalDelay|} 天"（绿色）
  - 第二行：`{totalChanges} 次变更 · {角色中文名×条数} · {角色中文名×条数}`（灰色辅助文字，按条数降序）。其中 `{totalChanges}` = 所有变更记录总数（含 daysDelta=0 的），`{角色×条数}` 仅统计 daysDelta ≠ 0 的变更（排除 reprioritize/pause/resume）。**supplement 类型**：daysDelta > 0 时按其 role 计入角色统计；daysDelta = 0 时不计入角色统计（但计入 totalChanges）
  - 第三行：`100% 来自需求变更 · 0 天来自开发`（红色强调）。当 totalDelay < 0 时改为"需求变更被工期节省抵消"

### 图表尺寸映射规则

**简洁版膨胀条**：

```
可用宽度 = 容器宽度 - 左右 padding（各 16px）
段天数总和 = originalTotalDays + Σ|daysDelta|（所有 daysDelta≠0 的变更）
基准天数 = max(originalTotalDays, currentTotalDays, 段天数总和)
每天像素 = 可用宽度 / 基准天数

原始计划条：宽度 = originalTotalDays × 每天像素
实际工期条：
  蓝色基底段 = originalTotalDays × 每天像素
  各变更段 = |daysDelta| × 每天像素，从左到右依次排列
  实际工期标记线 = currentTotalDays × 每天像素（竖线标注，与段宽度独立）
```

- 两条对齐左端，使用同一比例尺
- 实际工期条的段宽度可能超出 currentTotalDays 标记线（因非关键路径变更的输入量 > 实际影响），此时段在标记线右侧正常显示
- 最小段宽度 = 2px（防止极小段不可见）
- 段标注：段宽度 ≥ 40px 时内联显示文字，否则用连线标注到外部（标注位于条下方）

**详细版甘特条**：

```
X 轴 = 相对天数（Day 0 起始，非日历日期）
每天像素 = 可用宽度 / max(所有需求的 endDay)
每行：蓝色段 x=startDay×px, width=originalDays×px
      变更段紧接蓝色段后，width=|daysDelta|×px
```

- 需求行高 = 32px，行间距 = 8px
- 新增需求（isAddedByChange=true）整行用靛色（#5B21B6 / 导出 #5856D6）。若该需求后续有 add_days 变更，变更段仍按角色着色（不用靛色），仅原始天数部分为靛色
- 被砍需求整行灰色 + 删除线覆盖层
- 暂停需求整行虚线边框 + 浅灰填充
- **需求补充(supplement)**：daysDelta > 0 时在甘特条末尾显示虚线边框 + 玫瑰红浅色填充（#E11D48 opacity 20%）的延伸段，与 add_days 的实线段视觉区分；daysDelta = 0 时不渲染甘特段（仅在时间线中记录事件）

### 详细版（时间线+甘特）

**上部 — 时间线**：
```
04/01  PM 张三：RBAC权限 +4天
04/05  领导 王总：加数据大屏 +3天（新需求）
04/08  PM 李四：砍导出功能 -3天（节省）
04/10  PM 张三：用户管理要支持LDAP +1.5天（补充·条件变更）
```

**下部 — 甘特条**：
```
用户管理   ████████ ▓▓▓▓      原始8天 + 变更4天
数据大屏        ████████        新增需求（靛色）
导出功能   ████                 已砍（灰色删除线）
报表模块   ██████████          无变更
```

- 每个需求一行，按 sortOrder 排序（与需求列表一致）
- 蓝色 = 原始工期
- 变更增加部分颜色**按角色着色**（与简洁版一致：PM=红, 领导=橙, 测试=紫, 其他=灰）。**例外**：`supplement` 类型段使用玫瑰红（#E11D48），虚线边框+浅色填充，不按角色着色
- **同一需求多次变更**：变更段按 date 正序依次拼接在蓝色段后，每段独立着色。例：需求 A 原始 8 天 + PM 加 3 天(红) + QA 加 2 天(紫) → 蓝8 + 红3 + 紫2
- 灰色删除线 = 被砍需求
- 靛色整条 = 新增需求（isAddedByChange=true，整条靛色 #5B21B6 / 导出 #5856D6，不再按角色分色，与 QA 紫色 #7C3AED / #AF52DE 区分）
- 暂停/恢复事件显示在时间线中（标注"暂停"/"恢复"，甘特条中无天数变化段。daysDelta=0 的变更不渲染甘特段）
- 暂停的需求甘特条用虚线边框 + 浅灰填充标注"已暂停"
- **依赖关系**：MVP 不在甘特条间画箭头/连线（简化实现）。依赖信息仅在需求列表中以标签形式显示
- **底部汇总**（单行格式，与简洁版的三行格式不同——by design，详细版信息密度高，底栏精简）：`原计划 {originalTotalDays}天 → 实际 {currentTotalDays}天 · {changeCount}次变更 · {roleSummary} · 0天来自开发`。**负延期时**格式变为：`原计划 {X}天 → 实际 {Y}天 · 提前 {|delay|}天 · {changeCount}次变更 · {roleSummary} · 需求变更被工期节省抵消`
