# Scope Shield（工期盾）— 需求讨论记录

> 产品访谈过程中的所有确认决策，持续更新

## 产品定位

- **一句话**：给中小公司开发者，在项目中途被加/改需求时，用30秒记录变更并一键生成工期影响图，避免延期背锅
- **频率**：每周数次
- **主用户**：5-15人团队的前端/后端开发（有飞书但用得模糊，不清晰）
- **北极星任务**：记录需求变更 → 一键生成可视化图表 → 发群甩锅

## 核心输出（两种图混搭）

| 图表 | 用途 | 场景 |
|------|------|------|
| **A 膨胀对比条** | 日常发群，3秒看懂 | 简单场景 |
| **C 时间线故事+甘特条** | 完整证据链 | 多需求复杂场景 |

## 确认决策清单

### 基础设定
- [x] 时间单位：**天**
- [x] 平台形态：**Web 单页应用，免登录**
- [x] 数据存储：**本地优先（IndexedDB）**
- [x] 技术栈：**React + TypeScript + 自研 SVG 图表**

### 需求管理
- [x] 依赖关系：**简单依赖**（A完了才能做B，影响自动顺延）
- [x] 并行处理：**各需求独立计天，总工期=最长路径**
- [x] 变更类型：**全量** — 加天数 + 新增需求 + 砍需求 + 优先级调整 + 暂停/恢复
- [x] 优先级调整：**自动重排**，C提前则A/B自动后移，图表标注变更记录

### 依赖 & 顺延
- [x] A→B 有依赖，A 延期 → **B 自动往后推，提示用户"因A延期，B被顺延"**

### 暂停/恢复
- [x] **暂停期间不计入工期**，恢复后剩余天数继续倒数

### 砍需求
- [x] **灰色删除线划掉，工期回收显示为绿色"节省X天"**

### 责任人
- [x] 4 类角色：**产品经理（默认）/ 领导 / 测试 / 其他**
- [x] 可选填具体人名，系统自动记住
- [x] UI：下拉选角色 + 可选填人名

### 导出图片
- [x] 可选尺寸，**默认手机 390px 宽**，高度自适应
- [x] 长内容下拉展示
- [x] 导出风格：**Apple/iOS 亮色风格**（第一版鲜艳色，圆角、毛玻璃、透明）
- [x] 人名显示：**直接显示人名**

### App 界面风格
- [x] **中性风格（类似 Linear/Notion）**，导出图单独 Apple 亮色
- [x] 图表切换：**顶部 Tab "简洁版 | 详细版"**

### 项目管理
- [x] 归档：**手动归档 + 可恢复**
- [x] 多项目：**左侧项目列表，点击切换**
- [x] 首次体验：**预填 Demo 项目**
- [x] 项目开始日期：**默认今天，可修改**

### 需求录入
- [x] **逐条添加**：点"+ 添加需求"→ 填名称+天数（30秒核心）

### 变更记录
- [x] 变更描述：**必填，可以很短（一句话）**

### 数据安全
- [x] **导出 JSON 备份 + 一键导入恢复**

### 撤销/编辑
- [x] **每条变更可编辑或删除，删除时确认弹窗**

### V2 预留
- [ ] 云同步 / 团队协作
- [ ] 飞书数据拉取（后续从飞书导入需求）
- [ ] AI 工时估算
- [ ] 移动端适配
- [ ] 数据结构预留飞书相关字段
